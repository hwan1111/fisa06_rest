# 데이터 활용 DB 적재, 분석 및 시각화 진행 (데이터를 입력받고, 우리의 DB에 적재하고 해당 데이터에 적합한 방식으로 시각화한다)

## 프로젝트 개요

이 프로젝트는 **Streamlit**을 이용하여 데이터를 입력받고 이를 **DB**에 적재한 후, 적합한 방식으로 시각화하여 최종 사용자에게 제공합니다. 주어진 데이터셋을 사용하여 웹 애플리케이션을 구현하고, 분석된 데이터를 시각화하여 인사이트를 제공합니다.

### **프로젝트 진행 배경**

- **프로젝트 목표**:
  - 부트캠프 동료들 간에 **맛집 정보와 리뷰를 공유**할 수 있는 플랫폼을 구축
  - **맛집 미슐랭 가이드**는 **부트캠프 동료들이 서로 가본 맛집의 위치와 리뷰를 등록**하여 **서로의 경험을 공유**할 수 있는 공간을 제공

- **기획 배경**:
  - **맛집 추천과 후기**는 매우 유익하지만, 다양한 장소에 흩어져 있는 정보들을 **체계적으로 모은다면** 더 큰 가치를 제공할 수 있음
  - **서로의 경험을 실시간으로 공유**하고, 이를 **지도로 확인**할 수 있는 서비스 제공

- **서비스 기능**:
  - **맛집 찾기**: 지도에서 맛집을 손쉽게 찾을 수 있음
  - **카테고리별 추천**: 카테고리별로 추천된 맛집들을 한눈에 확인
  - **실시간 업데이트**: 사용자들이 제공하는 데이터는 실시간으로 업데이트됨
  - **위치 확인**: 지도 상에서 맛집의 정확한 위치를 확인 가능
  - **인기 맛집 파악**: 카테고리별로 인기 있는 맛집들을 확인

- **기술적 기여**:
  - 부트캠프에서 배운 **데이터 처리 및 시각화 기술**을 실제로 활용하는 좋은 기회

---

## 기술 스택

- **Streamlit**: 웹 애플리케이션의 인터페이스 구성
- **Pandas**: 데이터 분석 및 가공
- **MySQL**: 데이터 저장 및 관리
- **Folium**: 지리적 데이터 시각화
- **Plotly**: 인터랙티브 차트 시각화
- **Geopy**: 주소를 위도, 경도로 변환

## 사용자 요구사항 정의 및 개발 계획

### **사용자 요구사항 정의**

프로젝트에 대한 사용자 요구사항을 다음과 같이 정의했습니다:

1. **로그인 기능**:
   - **목표**: 사용자가 자신의 계정을 통해 로그인하여 개인화된 서비스를 이용할 수 있도록 한다.
   - **기능**: 사용자 ID와 비밀번호를 이용하여 로그인 기능을 구현한다. 로그인 후 사용자는 맛집을 등록하고, 리뷰를 작성하며, 다른 사용자의 리뷰를 볼 수 있다.

2. **맛집 등록 기능**:
   - **목표**: 사용자가 자신이 방문한 맛집에 대해 정보를 등록하고, 별점과 리뷰를 남길 수 있도록 한다.
   - **기능**: 맛집 이름, 주소, 카테고리, 별점, 방문 후기, URL(네이버 지도 링크) 등을 입력받고, 이를 Google Sheets나 MySQL에 저장한다.

3. **팀원 모집 기능**:
   - **목표**: 사용자가 다른 동료들과 함께 맛집을 방문할 수 있도록 팀원 모집 기능을 제공한다.
   - **기능**: 사용자는 자신이 방문할 맛집을 등록하고, 이를 보고 다른 동료들과 함께 갈 사람을 모집할 수 있다. 모집된 팀원들은 맛집을 함께 방문할 수 있다.

---

### **개발 계획**

1. **로그인 기능 구현**:
   - **기능 설명**: 사용자는 사용자 ID와 비밀번호로 로그인하며, 로그인 정보를 바탕으로 세션을 유지하여 개인화된 기능을 이용할 수 있다.
   - **개발 단계**:
     - 로그인 폼을 생성하여 사용자 ID와 비밀번호 입력을 받는다.
     - 로그인 정보를 `st.session_state`에 저장하여 다른 페이지에서 사용자 정보를 참조한다.
     - 로그인 후 사용자는 자신의 리뷰를 관리할 수 있다.

2. **맛집 등록 기능 구현**:
   - **기능 설명**: 사용자가 자신이 방문한 맛집 정보를 등록할 수 있다. 등록된 맛집 정보는 지도에 표시되며, 다른 사용자들이 이를 열람할 수 있다.
   - **개발 단계**:
     - Streamlit 폼을 사용하여 맛집 이름, 주소, 카테고리, 별점, 후기 등을 입력받는다.
     - **Geopy**를 사용하여 주소를 위도 및 경도 좌표로 변환하고, 이를 지도에 표시한다.
     - **MySQL**을 사용하여 등록된 데이터를 저장하고 관리한다.
     - 맛집 데이터를 **Folium**을 사용하여 지도 위에 시각화하고, **Plotly**를 이용해 별점 차트를 제공한다.

3. **팀원 모집 기능 구현**:
   - **기능 설명**: 사용자가 맛집 방문 시, 팀원 모집을 위한 기능을 제공한다. 다른 사용자는 모집된 팀원 정보를 확인하고 함께 방문할 수 있다.
   - **개발 단계**:
     - 각 맛집에 대해 팀원 모집을 위한 버튼을 제공한다.
     - **MySQL**을 사용하여 모집된 팀원 목록을 관리하고, 다른 사용자들이 해당 팀원 모집 정보를 확인할 수 있게 한다.
     - 사용자는 팀원 모집 게시판에서 맛집 방문 팀원을 모집하고, 참여할 수 있다.

## 아키텍처

애플리케이션은 크게 **프레젠테이션 계층(Presentation Layer)**, **애플리케이션 로직 계층(Application Logic Layer)**, **데이터 계층(Data Layer)**의 세 가지 주요 계층으로 나눠져 있습니다.

- **프레젠테이션 계층**: Streamlit을 사용하여 사용자와의 인터페이스를 담당합니다. 웹 페이지 상의 데이터 입력, 시각화된 지도 및 차트 표시 등이 이 계층에서 처리됩니다.
- **애플리케이션 로직 계층**: 사용자가 입력한 데이터를 처리하고, 이를 데이터베이스와 연동하며, 분석 및 시각화 작업을 수행합니다.
- **데이터 계층**: MySQL을 이용해 데이터를 안전하게 저장하고 관리하며, Google Sheets와의 연동을 통해 데이터의 실시간 처리가 가능하게 합니다.

---

## 프로젝트 구성 요소

1. **데이터 입력 및 처리**
   - 사용자로부터 입력을 받기 위한 Streamlit 폼과 사이드바 제공
   - 데이터를 Google Sheets나 MySQL에 저장
   - 입력 받은 데이터를 Pandas로 분석 및 처리

2. **시각화**
   - **Folium**을 사용하여 데이터를 지도에 시각화
   - **Plotly**를 이용하여 카테고리별 평균 별점 등을 시각화하는 차트 제공

3. **DB 연동**
   - **MySQL**과의 연동을 통해 데이터를 안전하게 관리

4. **Streamlit Application**
   - 사용자 인터페이스(UI) 제공
   - 입력된 데이터를 시각적으로 표현
   - 실시간 데이터 업데이트 기능

## 데이터셋

프로젝트에서 사용된 데이터는 다음과 같습니다:

1. **기존 데이터셋**: 수업 중 제공된 예시 데이터
   - 예시 데이터셋을 기반으로 DB에 데이터를 적재하고 분석을 수행

2. **팀에서 선정한 데이터셋**: 프로젝트 수행에 적합하다고 판단한 데이터
   - 주어진 데이터셋을 분석하여 통찰력 있는 정보를 도출

---

## 시연회 내용

1. **데이터 선정 배경**:
   - **MySQL**을 활용하여 대규모 데이터 처리 및 안정적인 데이터 관리를 구현했습니다.

2. **분석 기준**:
   - 사용자의 리뷰 데이터를 기반으로 평균 별점을 계산하고, 카테고리별 인기 식당을 분석했습니다.
   - 특정 지역 내의 인기 식당을 지도 상에서 시각화하여, 사용자들이 쉽게 찾아볼 수 있도록 했습니다.

3. **팀원 기여**:
   - **팀원 김동환**: 전체적인 아이디어 기초 프레임 개발 및 Streamlit 애플리케이션 구현
   - **팀원 김유정**: MySQL 데이터베이스 연동 및 데이터 분석, 가격 데이터 시각화
   - **팀원 권민석**: 로그인 기능 개발, Folium을 이용한 지도 깜빡임 문제 해결
   - **팀원 이주형**: 데이터베이스 구축 및 데이터 정제

4. **트러블슈팅**:
   - **문제 1**: MySQL 데이터베이스 연결 오류
     - 해결: `.env` 파일에 있는 데이터베이스 자격 증명을 확인하고 수정하여 연결 문제 해결.
   - **문제 2**: 주소 데이터 오류로 인한 지도 표시 오류
     - 해결: **Geopy**를 사용하여 부정확한 주소를 처리하고 지도에 정상적으로 표시되도록 개선.
